# Plan: generate-skills 스킬 구성

## Context

`agents/claude/skills/generate-skills/` 디렉토리에 스킬 생성용 스킬을 구성한다. 사용자가 "스킬 만들어줘", "새 스킬 추가" 등 요청 시, 폴더 구조 → 프론트매터 → 지시사항 → 검증까지 단계적으로 안내하는 워크플로우 스킬이다.

현재 `references/review-checklist.md`와 PDF 가이드가 이미 존재하며, 이를 유지하면서 새 파일들을 추가한다.

## 파일 생성 순서

### Step 1: references/ 상세 문서 4개 생성

**1-1. `references/skill-structure.md`** — 스킬 폴더/파일 구조 규칙
- 최소/확장 디렉토리 구조
- 폴더명 규칙 (kebab-case, name 필드와 일치)
- 각 하위 폴더(references/, scripts/, assets/) 용도
- Progressive Disclosure 원칙
- 금지 항목 (README.md, claude/anthropic 접두사)

**1-2. `references/frontmatter-spec.md`** — YAML frontmatter 필드 명세
- 필수 필드: name (kebab-case, 폴더명 일치), description (≤1024자, WHAT+WHEN, XML 태그 금지)
- 선택 필드: model, disable-model-invocation, license, metadata
- 구분자 규칙 (`---`)
- 유효성 검사 기준 체크리스트

**1-3. `references/patterns.md`** — 5가지 스킬 패턴 가이드
- 선형 워크플로우 / 인터뷰 기반 / 도구 오케스트레이션 / 템플릿 채우기 / 검증·리뷰
- 각 패턴별: 언제 사용하는가, 구조 예시, 장단점
- 패턴 선택 가이드 테이블

**1-4. `references/description-examples.md`** — description 작성 예시 모음
- WHAT + WHEN 공식 설명
- 좋은 예시 (영어/한국어 각각)
- 나쁜 예시 → 수정 예시
- 길이 최적화 팁
- 트리거 튜닝 전략 (과소/과잉)

### Step 2: scripts/validate-skill.sh 생성

검증 스크립트. 의존성: `yq` (brew install yq)

검사 항목:
1. SKILL.md 존재 (없으면 즉시 종료)
2. README.md 없음
3. YAML frontmatter `---` 구분자 존재
4. `name` 필드 존재 + kebab-case 정규식 매칭
5. `name` = 폴더명 일치
6. `description` 필드 존재 + ≤1024자
7. frontmatter 내 XML 태그 없음
8. `name`이 claude/anthropic 접두사 아님

설계 결정:
- `yq --front-matter=extract` 플래그로 마크다운 파일에서 YAML 직접 파싱
- 에러 누적 후 최종 요약 출력 (SKILL.md 부재 시에만 즉시 종료)
- 스크립트 초반에 `command -v yq` 체크 → 미설치 시 안내 메시지
- `chmod +x` 필요 → 생성 시 실행 권한 부여

### Step 3: SKILL.md 생성

```yaml
---
name: generate-skills
description: 새로운 Claude 스킬을 폴더 구조, 프론트매터, 지시사항까지 단계적으로 생성한다.
  스킬 만들어줘, 새 스킬 추가, SKILL.md 작성, generate-skills 요청 시 사용한다.
model: opus
disable-model-invocation: true
---
```

5단계 워크플로우:
1. **사용 사례 파악** — AskUserQuestion으로 문제/시나리오/도구/출력 파악, patterns.md에서 패턴 선택
2. **구조 생성** — skill-structure.md 참조하여 폴더/파일 생성
3. **프론트매터 작성** — frontmatter-spec.md + description-examples.md 참조
4. **지시사항 작성** — 선택된 패턴 구조로 본문 작성
5. **검증** — validate-skill.sh 실행 → 실패 시 해당 단계로 복귀 → review-checklist.md 최종 확인
